# Player QA Checklist (Founder Beta)

Run this with at least one real player account and one phone-size viewport.

## 1) Account and login

- [ ] Account creation works (if enabled in this environment).
- [ ] Login succeeds and session persists as expected.

## 2) Session join and reconnect

- [ ] Player can join using DM-provided session/invite details.
- [ ] Rejoin works after refresh/disconnect.
- [ ] Player returns with expected role/permissions.

## 3) Character and sheet basics

- [ ] Character/profile import works (if import path is enabled).
- [ ] Manual character setup/edit works.
- [ ] Player sheet opens and updates reliably.

## 4) Species/identity options

- [ ] Species selection/change works if supported by current build.
- [ ] Resulting sheet/stat display updates correctly.

## 5) Equipment and AC behavior

- [ ] Equipment can be added/removed.
- [ ] Equip/unequip updates expected fields.
- [ ] AC reflects equipment changes.

## 6) Actions/resources/spells

- [ ] Actions list is readable and usable.
- [ ] Resource tracking works for a basic flow.
- [ ] Spell interaction/display works for at least one spellcaster test character.

## 7) Combat visibility and turn context

- [ ] Initiative panel is visible and understandable to player.
- [ ] Current turn/round context is readable.

## 8) Phone-size usability

- [ ] Join/login flow works on phone.
- [ ] Core play interactions are usable at phone size.
- [ ] Major clutter/overlap/truncation issues captured as feedback.
