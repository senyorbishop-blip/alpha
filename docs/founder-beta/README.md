# Founder Beta QA + Feedback + Support Pack

Use this folder as the single operational pack for founder-beta testing.

## Start in this order

1. `master-checklist.md` — pre-send and pre-test readiness pass
2. `playtest-flow.md` — run a structured multi-role session
3. `dm-qa-checklist.md` / `player-qa-checklist.md` / `admin-qa-checklist.md`
4. `how-to-report-a-bug.md` + `bug-report-template.md`
5. `feedback-template.md` for non-bug product feedback
6. `triage-severity-guide.md` + `blocker-vs-non-blocker.md`
7. `support-flow.md` for intake, reproduction, labeling, and communication
8. `release-go-no-go-checklist.md` before each external send
9. `../known-issues-founder-beta.md` to communicate transparent system limits and founder-beta readiness

## Intended use

- Keep this process lightweight and repeatable.
- Prioritize multiplayer stability and role-based usability (DM/player/viewer/admin).
- Preserve known limitations with transparent communication instead of overpromising.
