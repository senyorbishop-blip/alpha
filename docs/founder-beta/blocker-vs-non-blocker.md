# Blocker vs Non-Blocker Guide (Founder Beta)

## Blocker (No-Go unless fixed)

Treat as release blocker when any of the following is true:

- Critical issue exists in install/start/join/save/combat core flows.
- No safe workaround exists for a High-severity issue affecting most testers.
- Admin recovery path (including password reset) is non-functional.
- Known issue creates unacceptable data-loss risk.

## Non-blocker (Can ship with known issue note)

Can proceed when:

- Issue is Medium/Low and workaround is clear.
- Issue affects a narrow edge case not in planned beta scenarios.
- Cosmetic/polish issue does not impair core play.

## Rule of thumb

If it prevents founder testers from completing a 60–90 minute session with DM + at least one player, treat it as blocker.
