# Founder Beta Structured Playtest Flow

Use this for an initial small-group founder beta cycle (DM + 1–3 players + optional viewer).

## Stage 1: First-session test (15-20 min)

Goal: Verify that people can get in and begin play.

- DM starts session and shares join info.
- Player joins from separate browser/device.
- Optional viewer joins with viewer role.
- Validate chat + token visibility + basic movement sync.

## Stage 2: First phone-user test (10-15 min)

Goal: Confirm phone-size baseline usability.

- One player joins from phone.
- Complete join/login, chat, and one simple gameplay interaction.
- Capture layout friction and input/scroll problems.

## Stage 3: First DM prep test (15-20 min)

Goal: Validate prep operations before live encounter.

- DM loads/imports map.
- DM configures fog and token placement.
- DM tests one walls/doors interaction.
- Player confirms expected visibility state.

## Stage 4: First combat test (15-20 min)

Goal: Confirm live round flow.

- DM starts combat.
- Players view initiative and complete at least one turn each.
- DM ends combat and verifies UI state recovers cleanly.

## Stage 5: Reconnect + save/load test (10-15 min)

Goal: Validate persistence and recovery.

- Trigger disconnect/refresh for DM and player.
- Verify rejoin behavior.
- Save state and reload.
- Confirm map/token/combat-relevant state is retained as expected.

## Exit criteria for this playtest run

- No unresolved Critical issues.
- All High issues have explicit workaround or immediate patch plan.
- Top 3 usability problems are documented for next patch cycle.
