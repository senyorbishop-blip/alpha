# DM Guide (Founder Beta)

## Start a session

1. Launch server
2. Open app URL in browser
3. Choose **Dungeon Master**
4. Start or enter a session

Keep note of:

- Session ID
- Player invite code
- Viewer invite code

## Core DM workflow

- Open/prepare map in the play page
- Place and organize tokens
- Use fog/vision tools for reveals
- Drive combat and narration/audio as needed

## Maps and editor basics

- Use map editor controls in `play.html` runtime
- Import/upload maps/assets through existing editor/map library surfaces
- Save map state before major session milestones

## Fog and vision

- DM controls fog reveal/hide
- Validate player perspective in a second tab/device when possible

## Combat operations

- Start combat from DM controls
- Track turn order and movement state
- Apply token HP/status/conditions as needed

## Save/load practice

- Runtime uses persistent data DB + map storage
- Perform regular backups before major map edits or release updates

## Multi-device notes

- **Desktop/laptop:** best for full DM workflow (editor + combat + map controls)
- **Tablet:** usable for moderate control tasks
- **Phone:** suitable for quick admin checks, not ideal for full prep/editing

## DM founder-beta tips

- Keep a player tab open for sync verification
- Announce known beta limitations to players up front
- Backup before update or large prep changes
